# 答辩问答预案（QA_PREP）

> 预测 15 个高概率提问 + 参考回答思路。答辩临场**不用背稿**，重要的是熟悉每个问题的 3--5 个关键词。

---

## A 组：数据与基线相关

### Q1. BUSI 只有 780 张，这么小的数据集结论是否可信？

**要点**：
- 小样本是医学影像的常态，而非本课题的特殊缺陷；
- 使用**分层抽样 + 固定种子 + 70/15/15 和 80/10/10 两种划分复核**，两种划分下结果均在 94--96% 区间；
- 宏平均 AUC = 99.63% 说明不是"蒙对"了少数样本；
- 并用 Grad-CAM 定性验证决策来源 = 病灶。

### Q2. 为什么不用外部数据集做交叉验证？

**要点**：
- 时间与数据获取权限限制；
- **已在"未来工作"中明确列出**——引入 UDIAT、BUS-BRA 等多中心数据做迁移验证；
- 本课题的定位是方法论验证，不是临床部署。

### Q3. 类别不平衡是如何量化处理的？

**要点**：
- 4 个层次互补：
  1. **WeightedRandomSampler**（数据层面均衡）；
  2. **Focal Loss**（Stage 4）+ **Label Smoothing**（Stage 5）在损失层面聚焦难样本；
  3. **评估层面**使用**宏平均**，避免多数类掩盖；
  4. **Stage 3/4 对比**专门分析了少数类的变化。

## B 组：方法选型相关

### Q4. 为什么最终用 EfficientNet-B0，而不是更大的 B3/B7 或 ResNet50？

**要点**：
- **参数效率**：B0 仅 400 万参数，比 ResNet18（1117 万）还少；
- **ImageNet 基准**：Top-1 从 69.3%（ResNet18）提升到 77.1%（B0）；
- **小样本防过拟合**：更大的 B3/B7 在 BUSI 780 张上容易过拟合；
- **推理速度**：Apple Silicon MPS 上实测单张 100--200 ms，满足 Web 交互；
- 对比实验未在论文中做，但属于经典选型结论。

### Q5. Focal Loss 和 Label Smoothing 能否共存？为什么 Stage 5 只用了 Label Smoothing？

**要点**：
- 理论上可以叠加，但二者**设计目标不同**：
  - Focal Loss 主要解决**类别不平衡**；
  - Label Smoothing 主要改善**模型校准**和**泛化**；
- Stage 5 已引入 **WeightedRandomSampler 做数据层面均衡**，不平衡压力大幅下降；
- 此时 Focal Loss 的价值边际减弱，Label Smoothing 在泛化与校准上的收益更大；
- 经验上二者叠加易导致训练不稳定，故只保留一个。

### Q6. DAGAN 合成数据会不会引入分布偏移？

**要点**：
- **会**，Stage 3 对正常类 -10% 就是明证；
- 本课题**未单独使用 DAGAN**，而是作为**混合策略**的一部分（Stage 4 起）；
- Stage 5 的 TTA 和标签平滑进一步降低了合成噪声的副作用；
- 未来工作计划用 Diffusion 替代 DAGAN 提升合成质量。

### Q7. 为什么不做语义分割 + 分类两阶段？

**要点**：
- BUSI 本身提供分割 mask，但**本课题专注于分类这一核心临床问题**；
- 端到端分类更贴合 Web 系统的交互延迟要求；
- Grad-CAM 已经提供了"软定位"能力，在可解释性上接近分割效果；
- 分割加分类两阶段属于扩展方向，纳入展望。

## C 组：结果与指标相关

### Q8. 验证集 98.31% 和测试集 94.87% 差了 3.44%，是否过拟合？

**要点**：
- 验证集和测试集都只有 117 张，**单张的预测差异就可能造成 1--2% 波动**；
- 3.44% 的差距在小样本统计意义下是合理的；
- **80/10/10 划分下 Val 与 Test 都是 96.15%**，说明划分随机性是主因；
- 早停 + 标签平滑已明显抑制过拟合（训练曲线未暴涨）。

### Q9. 为什么 TTA 反而比标准测试低 0.85%？

**要点**：
- EfficientNet-B0 训练时已充分使用翻转/旋转等在线增强，**模型本身已学到不变性**；
- TTA 的某些角度扰动可能让**原本预测正确的样本跨过阈值变成错误**；
- TTA 的真实价值是**提升鲁棒性和置信度校准**，而非绝对精度；
- 实际部署建议配合置信度阈值使用 TTA。

### Q10. 恶性 93.55% 够不够临床用？漏诊怎么办？

**要点**：
- 本系统定位是**"AI 第二意见"**，不是替代医生；
- Grad-CAM 热力图让医生能**一眼看出 AI 看哪里**，从而辅助决策而非盲信；
- Web 系统返回**完整概率分布**，医生可设置更低的恶性判定阈值降低漏诊；
- 临床部署前需跨中心验证与持续校准，**这在未来工作中已标注**。

## D 组：工程与系统相关

### Q11. 为什么采用前后端分离而非 Gradio/Streamlit？

**要点**：
- **职责清晰**：React 前端负责交互与可视化，Flask 专注推理与持久化；
- **可扩展**：未来接入 DICOM、权限、用户系统更方便；
- **工程化**：TypeScript 静态类型 + 组件化 UI，更接近真实产品；
- Gradio/Streamlit 适合 Demo，不适合作为毕业设计呈现的"系统级"工作。

### Q12. SQLite 会不会撑不住真实医院的并发？

**要点**：
- 当前定位是**单机原型**，SQLite 的读性能足够支撑本地 10--20 并发；
- 数据模型已抽象为 ORM，**切换到 PostgreSQL 只需改连接串**；
- 医院部署方案属于未来工作，需配合 DICOM 接入与权限体系。

### Q13. 推理速度 100--200 ms 是怎么测的？

**要点**：
- Apple Silicon M 系列 + PyTorch MPS 后端；
- 单张 300×300 图像经 EfficientNet-B0 前向传播 + Grad-CAM 渲染 + JSON 序列化全流程；
- 是**端到端的 API 延迟**，非单纯模型 forward 时间。

## E 组：深层 / 开放提问

### Q14. 如果让你重做，你会怎么改进？

**要点**：
- **数据侧**：引入多中心数据 + Diffusion 生成提升合成质量；
- **模型侧**：尝试 ViT 系列 + 自监督预训练（SimCLR/DINO）；
- **任务侧**：联合建模分割 + 分类 + BI-RADS 分级；
- **系统侧**：加入 DICOM 支持、医生批注回传形成闭环、做联邦学习避免数据出院。

### Q15. 这个工作的"不可替代性"是什么？开源模型直接微调能不能达到一样的效果？

**要点**：
- 本课题的**核心贡献不是 SOTA 数字，而是方法论**：
  - **五阶段递进式实验框架**——让每一步增益可归因；
  - **混合增强策略的系统化分析**——DAGAN 和传统增强的互补性被首次拆解；
  - **端到端交付链路**——从训练到可解释性再到可部署 Web，形成完整工程示例；
- 直接微调开源模型**可以复现数字**，但**无法复现方法论的价值**。

## 应对技巧

1. **听完整个问题再答**——不要抢话。
2. **用数字回答数字**——评委问数据，直接给数据；不绕弯。
3. **不知道就承认**——"这一点我目前确实没有深入研究，是一个很好的未来方向"比硬答失分少。
4. **发挥 Grad-CAM 这张牌**——任何涉及"可信度""可解释""医生怎么看"的问题都可以用它承接。
5. **用"未来工作"收场**——遇到超纲问题，导入展望中已列出的方向。
