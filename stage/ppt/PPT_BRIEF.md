# PPT 制作总体简报（Brief）

> 一页读懂：这份 PPT 要做成什么样。

## 1. 场景与目标

- **场景**：华中科技大学计算机学院本科毕业设计「最终答辩」。
- **受众**：3--5 位答辩评委老师（计算机/人工智能背景，无临床医学背景）。
- **核心目标**：用 12--15 分钟，讲清楚
  - 「做了什么」：构建了一套乳腺癌超声图像自动诊断系统；
  - 「怎么做的」：五阶段递进式实验 + 一套 Web 原型系统；
  - 「效果如何」：测试集准确率 **94.87%**，AUC **99.63%**；
  - 「有什么亮点」：混合增强策略、Grad-CAM 可解释性、端到端 Web 系统。

## 2. 视觉风格

| 维度 | 建议 |
| --- | --- |
| 整体风格 | 学术严谨、简洁冷静、信息密度适中 |
| 主色调 | 深蓝（#1F3A68）+ 白/浅灰 + 少量橙色（#E8823B）强调关键数据 |
| 字体 | 中文：思源黑体 / 微软雅黑；英文：Inter / Helvetica |
| 字号 | 标题 28--32pt；正文 18--22pt；图注 14--16pt |
| 图片占比 | 每页至少 1 个视觉元素（图/表/图标），避免纯文字 |
| 动画 | **不使用花哨动画**，最多用淡入/出现 |

## 3. 页数与节奏

- **总页数 20--24 页**（含封面、目录、致谢）。
- **口播 12--15 分钟**，平均每页约 40--60 秒。
- 章节节奏建议：
  1. 封面 + 目录：1 分钟
  2. 研究背景与问题：1.5 分钟
  3. 技术方案与系统设计：3 分钟
  4. 五阶段实验与结果：4 分钟（**重点**）
  5. Grad-CAM 可解释性：1 分钟
  6. Web 诊断系统演示：2 分钟（**亮点**）
  7. 创新点与不足展望：1 分钟
  8. 致谢：0.5 分钟

## 4. 必须传达的 6 个关键点

1. BUSI 数据集规模小（780 张）、类别不平衡（56:27:17），这是核心挑战。
2. 设计了 **五阶段递进式控制变量实验**，使每一步改进可归因。
3. **DAGAN 合成数据 + 传统增强** 的混合策略互补性地提升不同类别识别率。
4. 最终方案（EfficientNet-B0 + 全套优化）**测试准确率 94.87%、AUC 99.63%**。
5. Grad-CAM 热力图验证模型决策依据 = 病灶区域，具备临床可解释性。
6. 交付了一套**前后端分离的 Web 原型系统**，支持单张/批量/历史三种模式。

## 5. 交付边界

**PPT 专家需要做的**：
- 按 `SLIDE_DECK_SPEC.md` 逐页生成 `.pptx`。
- 确保 `assets/images/` 中每张图都在合适的页面出现。
- 统一视觉与排版规范。

**PPT 专家不要做的**：
- 不要自行新增未在 `SLIDE_DECK_SPEC.md` 中列出的实验数据或方法描述。
- 不要改动数据（尤其是准确率、AUC 等数字）。
- 不要添加评审老师个人信息、政治相关内容。
